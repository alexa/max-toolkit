virtual void multiAgentExperience::experience::ExperienceObserverInterface::onExperienceChange ( const std::vector< Experience > &  experiences)
pure virtual

When the active experiences the device Ux should display change, this callback will be called.

Your implementation should return quickly from this callback. Ie, if long operations are required, store the list of experiences and perform that work an a different thread.

Parameters
experiencesThe list of currently active experiences. In priority order eg. experiences[0] is higher priority to display than experiences[1]
